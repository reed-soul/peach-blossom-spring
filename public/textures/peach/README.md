# Peach texture assets

This directory holds the PBR texture set for the peach-blossom tree foliage.
Generated by `scripts/texture/*.mjs` (ported from SeedThree, MIT) from a source
albedo image with transparency.

## Required files

| File | Source | Purpose |
|---|---|---|
| `peach_blossom.png` | **YOU PROVIDE** — a peach petal image with alpha (transparent background) | albedo/base color |
| `peach_blossom_normal.png` | `derive-pbr.mjs` derives it | tangent-space normals (Sobel) |
| `peach_blossom_roughness.png` | `derive-pbr.mjs` derives it | roughness map |
| `peach_blossom_translucency.png` | `derive-translucency.mjs` derives it | SSS thickness (edges transmit, veins opaque) |
| `peach_bark.png` | optional — bark albedo | trunk/branches |
| `peach_bark_normal.png` / `peach_bark_roughness.png` | `derive-pbr.mjs` | bark PBR |

## How to generate

1. Drop a peach petal photo here as `peach_blossom.png`. Best results:
   - Transparent background (PNG with alpha). If you only have a flat-background
     photo, mask it in Photoshop/GIMP first, OR use `chroma-key.mjs` (from
     SeedThree) if the background is solid magenta.
   - Square-ish, ~256–512 px per petal. The petal should fill most of the frame.
   - Pink / white / magenta petals all work; the GLSL tints per-instance.
2. From the project root:

   ```bash
   npm run texture:peach
   ```

   This runs dilate-alpha (kills bilinear dark halos) → derive-pbr (normal +
   roughness) → derive-translucency (SSS thickness).

3. (Optional) Repeat for bark: drop `peach_bark.png`, then

   ```bash
   node scripts/texture/derive-pbr.mjs public/textures/peach/peach_bark.png --strength 3.0
   ```
